# OpenAI 兼容性探测矩阵（HeyMean）

> 最后更新：2026-05-19
> 
> 本文档用于指导 `设置 > OpenAI 兼容` 的“连接测试”结果解读，帮助用户减少盲调。

## 1) 常见能力维度

- `/models`：用于拉取模型列表与连通性检查。
- `/chat/completions`：HeyMean 当前主通道。
- `/responses`：新一代统一接口，部分上游尚未实现。
- `tools`：函数调用能力（函数参数 JSON Schema）。
- `stream`：SSE 流式输出能力。

## 2) 常见上游的经验性支持情况（仅作参考）

| 上游类型 | `/models` | `/chat/completions` | `/responses` | tools | stream | 备注 |
|---|---|---|---|---|---|---|
| OpenAI 官方 | ✅ | ✅ | ✅ | ✅ | ✅ | 一般无需代理 |
| Azure OpenAI（OpenAI 协议） | ✅* | ✅ | ⚠️（版本相关） | ✅ | ✅ | 需关注 API version / deployment |
| 反向代理网关（OpenAI-compatible） | ✅/❌ | ✅ | ⚠️/❌ | ⚠️ | ✅/⚠️ | 经常出现部分能力缺失 |
| 本地推理网关（如 Ollama/LM Studio 类兼容层） | ✅/⚠️ | ✅/⚠️ | ❌/⚠️ | ⚠️/❌ | ✅/⚠️ | 接口可能“看起来兼容”但细节不全 |

> `✅*`：可用但路径或参数命名可能不同。

## 3) 连接测试策略（只读探测）

HeyMean 的“连接测试”遵循只读原则，不写入任何数据：

1. `GET /models`：验证鉴权、基址路径与最小连通性。
2. `POST /chat/completions`（极小请求）：验证主协议可用性。
3. `POST /responses`（极小请求）：验证新协议可用性。
4. （可选）携带最小 `tools` 参数进行一次兼容探测。
5. （可选）`stream=true` 探测响应头/返回形态是否为流。

## 4) 建议配置映射

- 若 `/chat/completions` 成功：建议协议模式设为 **Chat Completions**。
- 若 `/chat/completions` 失败但 `/responses` 成功：建议协议模式设为 **Responses**。
- 若 `/models` 失败但其它成功：建议检查 base URL 路径或网关路由改写。
- 若 tools 失败：建议关闭工具调用或切换支持 function-calling 的模型。
- 若 stream 失败：建议关闭流式输出（或通过 proxy 做 SSE 透传）。
- 若跨域/CORS 错误：建议启用反向代理（同源转发）。

## 5) 典型故障与提示语

- `401/403`：API Key 无效、权限不足或配额不足。
- `404`：端点路径不兼容（常见于缺少 `/v1` 或路由前缀）。
- `405`：方法不被允许（网关仅开放部分方法）。
- `429`：限流，建议稍后重试或降低并发。
- `5xx`：上游不稳定，建议切换节点或重试。
- 网络/CORS：前端直连受限，建议 proxy。

## 6) 与产品实现的关系

- 设置页“连接测试”会将探测结果转成人类可读建议文案。
- 文案重点：**该怎么改配置**，而不仅是报错码。
